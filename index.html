二、OpenFace

时间： 说明： 概况：

OpenFace is a Python and Torch implementation of face recognition with deep neural networks and is based on the CVPR 2015 paper FaceNet: A Unified Embedding for Face Recognition and Clustering by Florian Schroff, Dmitry Kalenichenko, and James Philbin at Google. Torch allows the network to be executed on a CPU or with CUDA.

  • 以上介绍来自于OpenFace官方论文：https://cmusatyalab.github.io/openface/

整体流程图：

1、Input Image -> Detect

  • 使用dlib 或 OpenCV中预先训练的模型检测面部 ;

  • 这一步使用的是传统的计算机视觉的方式，目标就是得到人脸具体的矩形框。

2、Detect -> Transform（仿射变换） -> Crop

  • 检测人脸中的关键点，然后根据这些关键点对人脸做对齐校准。
  • 使用dlib的 实时姿势估计 和OpenCV的 仿射变换 来尝试使眼睛和下唇出现在每个图像上的相同位置。
  • Face Alignment

3、Crop -> Representation

使用VGG16

  • 使用深度神经网络在128维单位超球面上表示（或嵌入）面部。嵌入是任何人脸上的通用表示。与其他面部表示不同，这种嵌入具有良好的特性，即两个面嵌入之间的较大距离意味着面部可能不是同一个人。此属性使得聚类，相似性检测和分类任务比其他特征之间的欧几里德距离无意义的人脸识别技术更容易。
  • 使用softmax作为损失函数，输出的结果不满足人脸识别的欧几里得度量要求。所以这里推荐使用center loss、triplet loss

4、Application

  • 最后得到了是一个向量，任务结束。
  • 将您喜欢的聚类或分类技术应用于功能以完成识别任务。请参阅下面的分类和相似性检测示例，包括在线网络演示。

原文链接：https://cmusatyalab.github.io/openface/

参考链接：https://www.zhihu.com/question/60759296

[pytorch]二、分类问题

问题：有3堆数据，每个数据的特征都是（x，y），在坐标系中可以对应显示，如何进行分类？

问题分析：

1、输入的数据为（x，y），说明输入层有2个特征，则对应的n_feature=2，2个特征；

2、输出的分类要分3类，说明输出层有3个特征，则对应的n_output=3，3个特征；

3、其中的n_hidden，不受影响，之前在回归问题中使用的是10个神经元的隐藏层，这里可以不做改变

细节介绍：

1、分类问题的损失函数用torch.nn.CrossEntropyLoss()，其计算的是softmax，即预测误差为【0.1,0.2,0.7】，最后在计算结果中的标签误差为【0,0,1】，即说明分类中属于第三种情况的可能性最大，将其分入第三类中。

2、输出格式的改变：
在回归问题中是：

y_pred=net(x)

  • 这里的输出是一个固定的数值，就是y的预测值

在分类问题中是：

out=net(x)

  • 这里的输出是一个列表，在这里是比如【10,23,77】这样的格式，所以用softmax()来进行转换，输出变为【0.1,0.2,0.7】这样的格式，即说明属于第3类的可能性最大

[人脸识别]Haar Cascade

1、Haar Cascade

时间：2001年。

说明：最经典的算法，被Matlab和OpenCV引用作为库。

概况：基于Haar特征的cascade分类器。它是一种机器学习方法，通过许多正负样例中训练得到cascade方程，然后将其应用于其他图片。

前言：

目标检测的基本原理（yolo之前的那些方案）：

1、先通过训练集学习一个分类器；

2、然后在测试图像中以不同scale的窗口滑动扫描整个图像；

3、每次扫描做一下分类，判断一下当前的这个窗口是否为要检测的目标。

  • 算法的核心是分类，分类的核心一个是用什么特征，一个是用哪种分类器。
  • Haar特征/矩形特征，计算Haar特征个数，是使用黑色的所有像素值和减去白色的所有像素值和，24x24尺寸的滑动窗口特征数就超过160000。

  • Integral Image，来重新表示一幅图像，把计算量一下就缩减了一个量级。

  • AdaBoost分类器，是把多个弱分类器结合起来形成一个强分类器，是这些弱分类器的加权和。每个弱分类器的准确率并不高，但最后形成的强分类器的准确率就很高了，有很好的泛化能力，从超过16万个的特征中选择最好的特征，只需要200个特征的分类器在检测中的精确度达到了95%。

    ”因此，我们将每个特征都应用于所有训练集图片中，对于每个特征，找出人脸图片分类效果最好的阈值。显然，分类会有误分类，我们选择分类错误率最小的那些特征，也就是说这些特征可以最好的将人脸图片和非人脸图片区分开。(实际上没有那么简单，每个图片初始时给定相同的权重。每次分类后，提升被分类错误图片的权重。然后再根据新的权重分类，计算出新的错误率和新的权重，直到错误率或迭代次数达到要求即停止。“

  • Cascade 分类器，有人后续引入，提出级联分类器的方法，窗口在第一个阶段就检测失败了，那么就直接舍弃它，无需考虑剩下的特征。 级联分类器的关键是强分类器的次序安排：把简单的、False Negative低的安排在最前面。 这样可以快速判断出那些不相关的区域，把更多的计算量用在潜在区域.
    • False Positive：把合法的判断成非法的，译为“误报”。（相当于医学上的假阳性）
    • False Negative：把非法的判断成合法，译为“漏报”。（相当于医学上的假阴性）

《Rapid Object Detection using a Boosted Cascade of Simple Features》
原文链接：https://ieeexplore.ieee.org/document/990517
参考链接：https://blog.csdn.net/u012116229/article/details/45057937

[c++]c++笔记记录

存储变量类型：

1、register 存储类用于定义存储在寄存器中而不是 RAM 中的局部变量，且不能对它应用一元的 ‘&’ 运算符。

2、static 修饰局部变量可以在函数调用之间保持局部变量的值。
static 修饰符也可以应用于全局变量。当 static 修饰全局变量时，会使变量的作用域限制在声明它的文件内。

3、extern 是用来在另一个文件中声明一个全局变量或函数。
无法初始化的变量。
全局变量对所有的程序文件都是可见的。

函数参数：

调用类型 描述
传值调用 修改函数内的形式参数对实际参数没有影响。
指针调用 该方法把参数的地址复制给形式参数，修改形式参数会影响实际参数。
引用调用 该方法把参数的引用复制给形式参数，修改形式参数会影响实际参数。

字符串：

指针：

动态内存分配。
指针是一个变量，其值为另一个变量的地址。
所有指针的值的实际数据类型，不管是整型、浮点型、字符型，还是其他的数据类型，都是一样的，都是一个代表内存地址的长的十六进制数。
定义一个指针变量->把变量地址赋值给指针->访问指针变量中可用地址的值。
