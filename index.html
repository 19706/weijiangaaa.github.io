[pytorch]二、分类问题

二、分类问题

问题：有3堆数据，每个数据的特征都是（x，y），在坐标系中可以对应显示，如何进行分类？

问题分析：

1、输入的数据为（x，y），说明输入层有2个特征，则对应的n_feature=2，2个特征；

2、输出的分类要分3类，说明输出层有3个特征，则对应的n_output=3，3个特征；

3、其中的n_hidden，不受影响，之前在回归问题中使用的是10个神经元的隐藏层，这里可以不做改变

细节介绍：

1、分类问题的损失函数用torch.nn.CrossEntropyLoss()，其计算的是softmax，即预测误差为【0.1,0.2,0.7】，最后在计算结果中的标签误差为【0,0,1】，即说明分类中属于第三种情况的可能性最大，将其分入第三类中。

2、输出格式的改变：
在回归问题中是：

y_pred=net(x)

  • 这里的输出是一个固定的数值，就是y的预测值

在分类问题中是：

out=net(x)

  • 这里的输出是一个列表，在这里是比如【10,23,77】这样的格式，所以用softmax()来进行转换，输出变为【0.1,0.2,0.7】这样的格式，即说明属于第3类的可能性最大

[linux]基础指令

2、Linux基本操作

以下都是从接触Jetson Nano开始，学习Linux的笔记：

2.1简介

Jetson Nano官方提高的系统镜像中预置了基于Ubuntu16.04的LTS系统，其中包含了吧啦吧啦一堆（）
这里是一些常见的问题说明，新手需要注意：

  • Linux下的文件，没有拓展名的概念，所以看到txt不要想当然的就认为是一个文本文件；
  • 同时，文件名是区分大小写的；

    2.2基础常用指令

    2.2.1 文件管理指令

    cd ~ #切换至home目录
    pwd #查看当前的工作目录
    cd / #切换至root目录
    cd .. #回到上一次目录

ls
ls f* #查看以f开头的所有文件和子目录
ls -a #查看包含隐藏文件的文件和子目录

cp 1.txt 2.txt #将文本1复制成文本2

mv

rm

2.2.2 编辑文本文件

通过nano编辑器来编辑文本文件

sudo nano 1.txt

查看文本文件内容

cat 1.txt
more 1.txt

将命令行文件导至文本文件中

echo “hello world” > 1.txt

2.2.2 目录管理

创建目录

mkdir

删除目录

rm

2.2.3 文件权限

查看文件权限

  • -rwx rwx rwx
    代表：文件；所有者可读、可写、可运行；群组。。。；其他用户。。。；

    ls -l

修改文件权限

chmod 777 1.txt

其中的(421)(421)(421)->777

变更文件所有者与群组
文本1的所有者变更为tina

chown tina 1.txt
文本1的所有者变更为root，但文件群组变更为tina
chown root:tina 1.txt

2.2.4 系统管理

超级用户权利来运行工作

sudo

修改密码

passwd

2.2.5 升级操作系统

2.2.6 安装软件套件

sudo apt-get install

2.2.7 自动运行程序

修改【/etc/rc.local】文件，在文件内容的最后加入以下信息：

其中【&】表明以后台方式运行程序

[git]常用命令

Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。

个人目前的常用途：
1、目前绝大多数优秀项目的源码都会开源在github上，为了clone；
2、个人博客是搭载github上的，git是基础命令工具，同是为的是素材的网络地址；
3、为了项目源代码的开源共享。

git log –oneline

回到过去

reset是在commit之间穿梭

git reset
git reset –hard HEAD
git reset –hard HEAD^^
git reset –hard HEAD~2

使用ID号码

git reset –hard 283968a

回到未来

查看“未来的ID号码”

git reflog

对单个文件进行回到过去

  • 说明：【checkout】是针对单个文件进行的回退，相比于【reset】，【checkout】操作后log中的所有记录均不发生改变；而【reset】后，整个log发生了改变，其下的【checkout】也会不同。

  • 比喻：【reset】改变了历史，包括其中的人物命运也随历史的不同而不同；【checkou】不改变历史，而是可以单独可查个人在不同版本历史的命运。就好像是针对历史中某个人物的命运改变，即只回退了单一文件的版本，

    git checkout 【ID】 – 1.py

[人脸识别]方法总结和对比分析

1、Haar Cascade

时间：2001年。

说明：最经典的算法，被Matlab和OpenCV引用作为库。

概况：基于Haar特征的cascade分类器。它是一种机器学习方法，通过许多正负样例中训练得到cascade方程，然后将其应用于其他图片。

前言：

目标检测的基本原理（yolo之前的那些方案）：

1、先通过训练集学习一个分类器；

2、然后在测试图像中以不同scale的窗口滑动扫描整个图像；

3、每次扫描做一下分类，判断一下当前的这个窗口是否为要检测的目标。

算法的核心是分类，分类的核心一个是用什么特征，一个是用哪种分类器。

  • Haar特征/矩形特征，计算Haar特征个数，是使用黑色的所有像素值和减去白色的所有像素值和。

!()[https://img-my.csdn.net/uploads/201211/23/1353646276_1594.png]

  • Integral Image来重新表示一幅图像，把计算量一下就缩减了一个量级。

  • Cascade 分类器，提出级联分类器的方法，检测一个窗口未必要计算所有这T个特征。

《Rapid Object Detection using a Boosted Cascade of Simple Features》
原文链接：(https://ieeexplore.ieee.org/document/990517)[https://ieeexplore.ieee.org/document/990517]

[c++]c++笔记记录

存储变量类型：

1、register 存储类用于定义存储在寄存器中而不是 RAM 中的局部变量，且不能对它应用一元的 ‘&’ 运算符。

2、static 修饰局部变量可以在函数调用之间保持局部变量的值。
static 修饰符也可以应用于全局变量。当 static 修饰全局变量时，会使变量的作用域限制在声明它的文件内。

3、extern 是用来在另一个文件中声明一个全局变量或函数。
无法初始化的变量。
全局变量对所有的程序文件都是可见的。

函数参数：

调用类型 描述
传值调用 修改函数内的形式参数对实际参数没有影响。
指针调用 该方法把参数的地址复制给形式参数，修改形式参数会影响实际参数。
引用调用 该方法把参数的引用复制给形式参数，修改形式参数会影响实际参数。

字符串：

指针：

动态内存分配。
指针是一个变量，其值为另一个变量的地址。
所有指针的值的实际数据类型，不管是整型、浮点型、字符型，还是其他的数据类型，都是一样的，都是一个代表内存地址的长的十六进制数。
定义一个指针变量->把变量地址赋值给指针->访问指针变量中可用地址的值。
