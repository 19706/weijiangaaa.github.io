gcc、clang和make、cmake

gcc、clang和make、cmake

  • 首先说明一下整个编程的过程是：
    源代码（.cpp）->编译程序->目标程序（.obj）->链接程序（整合目标程序.obj和库程序.lib）->可执行程序（.exe）。

  • 我们常使用的IDE，例如VS、Pycharm等等，通过“运行（F5）”，便可一步完成以上所有的工作，但是在其他环境中，例如Ubuntu、Mac OS中，我们为了使用更加轻量的开发环境，抛去冗杂的大型开发环境，在Vim以及Nano中，没有如此系统的“一键完成”，所以我们还是有必要学习以上的过程。

  • 其中完成第一步编译这个过程就是由gcc或者clang编译器来完成的，从源代码得到目标程序。这两者是不同的编译器，其中gcc速度较快，而且报错提示更加人性化。
  • 一般我们对于单一的源代码文件可以通过gcc完成编译，但是面对工程文件包含多个源代码文件的时候，挨个文件使用gcc自然麻烦而且效率低。
    而make借用makefile文件里的相关指令，例如常见的gcc -o test.c，来完成对所有源代码的编译，提高了编译的效率，一般只需要make指令，便可以完成所有的编译工作。
  • 以上，我们需要编写makefile，完成make需要的指令，这里makefile较复杂，所以我们一般都过cmake来完成makefile的自动生成，而生成的法则则是cmakelists.txt，相较于makefile，cmakelists.txt更加符合我们的编写习惯而且更加高级易读。

[人脸识别]二、OpenFace

检测MTCNN

识别FaceNet

OpenFace（OpenCV、dlib）

二、OpenFace

时间： 说明： 概况：

OpenFace is a Python and Torch implementation of face recognition with deep neural networks and is based on the CVPR 2015 paper FaceNet: A Unified Embedding for Face Recognition and Clustering by Florian Schroff, Dmitry Kalenichenko, and James Philbin at Google. Torch allows the network to be executed on a CPU or with CUDA.

  • 以上介绍来自于OpenFace官方论文：https://cmusatyalab.github.io/openface/

整体流程图：

1、Input Image -> Detect

  • 使用dlib 或 OpenCV中预先训练的模型检测面部 ;

  • 这一步使用的是传统的计算机视觉的方式，目标就是得到人脸具体的矩形框。

2、Detect -> Transform（仿射变换） -> Crop

  • 检测人脸中的关键点，然后根据这些关键点对人脸做对齐校准。
  • 使用dlib的 实时姿势估计 和OpenCV的 仿射变换 来尝试使眼睛和下唇出现在每个图像上的相同位置。
  • Face Alignment

3、Crop -> Representation

使用VGG16

  • 使用深度神经网络在128维单位超球面上表示（或嵌入）面部。嵌入是任何人脸上的通用表示。与其他面部表示不同，这种嵌入具有良好的特性，即两个面嵌入之间的较大距离意味着面部可能不是同一个人。此属性使得聚类，相似性检测和分类任务比其他特征之间的欧几里德距离无意义的人脸识别技术更容易。
  • 使用softmax作为损失函数，输出的结果不满足人脸识别的欧几里得度量要求。所以这里推荐使用center loss、triplet loss

4、Application

  • 最后得到了是一个向量，任务结束。
  • 将您喜欢的聚类或分类技术应用于功能以完成识别任务。请参阅下面的分类和相似性检测示例，包括在线网络演示。

原文链接：https://cmusatyalab.github.io/openface/

参考链接：https://www.zhihu.com/question/60759296

[pytorch]二、分类问题

问题：有3堆数据，每个数据的特征都是（x，y），在坐标系中可以对应显示，如何进行分类？

问题分析：

1、输入的数据为（x，y），说明输入层有2个特征，则对应的n_feature=2，2个特征；

2、输出的分类要分3类，说明输出层有3个特征，则对应的n_output=3，3个特征；

3、其中的n_hidden，不受影响，之前在回归问题中使用的是10个神经元的隐藏层，这里可以不做改变

细节介绍：

1、分类问题的损失函数用torch.nn.CrossEntropyLoss()，其计算的是softmax，即预测误差为【0.1,0.2,0.7】，最后在计算结果中的标签误差为【0,0,1】，即说明分类中属于第三种情况的可能性最大，将其分入第三类中。

2、输出格式的改变：
在回归问题中是：

y_pred=net(x)

  • 这里的输出是一个固定的数值，就是y的预测值

在分类问题中是：

out=net(x)

  • 这里的输出是一个列表，在这里是比如【10,23,77】这样的格式，所以用softmax()来进行转换，输出变为【0.1,0.2,0.7】这样的格式，即说明属于第3类的可能性最大
