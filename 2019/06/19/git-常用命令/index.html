[git]常用命令

Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。

个人目前的常用途：
1、目前绝大多数优秀项目的源码都会开源在github上，为了clone；
2、个人博客是搭载github上的，git是基础命令工具，同是为的是素材的网络地址；
3、为了项目源代码的开源共享。

git log –oneline

回到过去

reset是在commit之间穿梭

git reset
git reset –hard HEAD
git reset –hard HEAD^^
git reset –hard HEAD~2

使用ID号码

git reset –hard 283968a

回到未来

查看“未来的ID号码”

git reflog

对单个文件进行回到过去

  • 说明：【checkout】是针对单个文件进行的回退，相比于【reset】，【checkout】操作后log中的所有记录均不发生改变；而【reset】后，整个log发生了改变，其下的【checkout】也会不同。

  • 比喻：【reset】改变了历史，包括其中的人物命运也随历史的不同而不同；【checkou】不改变历史，而是可以单独可查个人在不同版本历史的命运。就好像是针对历史中某个人物的命运改变，即只回退了单一文件的版本，

    git checkout 【ID】 – 1.py

Branch

Master主要给用户使用，其他的branch的话可以用作自己的开发

通过图表的方式来查看branch

git log –oneline –graph

建立分支

git branch dev

查看分支

git branch
其中带有【 * 】的说明是所处的分支

转换分支

gir checkout dev

删除分支

git checkout branch
git branch -d dev
先切换回主分支再进行其他分支的删除

建立分支2

git checkout -b dev

git commit -am “change”
前提是已经stage，已经在管理库中存在

转换主分支

git checkout master
进入master之后，branch不影响master，用户看到的还是master的

开发版本完成之后，将其从branch推到master中
确保进入了master

git merge dev
这条指令就在master中，将dev的信息推进来

加入信息，选项的内容为：

git merge –no-ff -m “keep me merge info” dev
